# RealProblem 部署指南

本指南面向非技术背景用户，一步步教你如何把 RealProblem 网站部署上线。

---

## 第一步：准备工作

你需要：
1. 一台电脑（Mac 或 Windows 都行）
2. 安装 Node.js（去 https://nodejs.org 下载"LTS"版本，一路下一步安装即可）
3. 一个 GitHub 账号（去 https://github.com 免费注册）

## 第二步：在本地运行

1. 打开电脑的"终端"（Mac）或"命令提示符"（Windows）

2. 进入项目文件夹：
```bash
cd 你的路径/RealProblem/realproblem-site
```

3. 安装依赖：
```bash
npm install
```

4. 初始化数据库：
```bash
npx prisma migrate dev --name init
```

5. 启动开发服务器：
```bash
npm run dev
```

6. 打开浏览器访问 http://localhost:3000 ，你应该能看到 RealProblem 的首页了！

## 第三步：配置 LLM API 密钥

要让 LLM Judge 评审功能工作，你需要获取三个 API 密钥：

### OpenAI (GPT)
1. 去 https://platform.openai.com 注册账号
2. 点击右上角头像 → "API Keys" → "Create new secret key"
3. 复制密钥

### Anthropic (Claude)
1. 去 https://console.anthropic.com 注册账号
2. 点击 "API Keys" → "Create Key"
3. 复制密钥

### Google AI (Gemini)
1. 去 https://aistudio.google.com/apikey 登录 Google 账号
2. 点击 "Create API Key"
3. 复制密钥

### 填入密钥

打开项目根目录下的 `.env` 文件，把密钥填进去：

```
DATABASE_URL="file:./dev.db"
OPENAI_API_KEY="sk-xxxxxxxxxxxxxxx"
ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxxxx"
GOOGLE_AI_API_KEY="AIzaxxxxxxxxxxxxxxx"
```

保存后重启开发服务器。

> 注意：不需要三个都有。有一个就能工作（只是评审只有一个模型的分数，而不是三个取中位数）。

## 第四步：部署到互联网

### 方案 A：Vercel 部署（推荐，免费）

**Vercel** 是一个免费的网站托管服务，专门为 Next.js 优化。

1. 先把代码上传到 GitHub：
   - 在 GitHub 上创建一个新仓库（New Repository），名字叫 `realproblem`
   - 在终端执行：
   ```bash
   cd 你的路径/RealProblem/realproblem-site
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/realproblem.git
   git push -u origin main
   ```

2. 去 https://vercel.com 用 GitHub 账号登录

3. 点击 "Add New Project" → 选择你刚创建的 `realproblem` 仓库 → "Import"

4. 在 "Environment Variables" 中添加你的 API 密钥：
   - `DATABASE_URL` = `file:./dev.db`
   - `OPENAI_API_KEY` = 你的密钥
   - `ANTHROPIC_API_KEY` = 你的密钥
   - `GOOGLE_AI_API_KEY` = 你的密钥
   - `CRON_SECRET` = 随便写一串密码（用于定时任务验证）

5. 点击 "Deploy"

6. 等待一两分钟，部署完成后会给你一个网址，如 `realproblem.vercel.app`

> **重要提示：** Vercel 免费版使用 Serverless Functions，SQLite 在这个环境下不能持久化数据。如果你需要数据持久化，请参考方案 B 使用外部数据库。

### 方案 B：VPS 部署（需要一点技术基础）

如果你有一台云服务器（如阿里云、腾讯云），可以直接部署：

```bash
# 在服务器上
git clone https://github.com/你的用户名/realproblem.git
cd realproblem
npm install
npx prisma migrate deploy
npm run build
npm start
```

建议配合 `pm2` 做进程管理，配合 Nginx 做反向代理。

### 方案 C：升级到 PostgreSQL + Supabase（推荐用于正式运营）

当你的投稿量变大后，建议从 SQLite 迁移到 PostgreSQL：

1. 去 https://supabase.com 注册免费账号
2. 创建一个新项目，获得数据库连接字符串
3. 修改 `prisma/schema.prisma` 中的 `provider = "sqlite"` 为 `provider = "postgresql"`
4. 更新 `.env` 中的 `DATABASE_URL` 为 Supabase 提供的连接字符串
5. 运行 `npx prisma migrate dev --name switch-to-postgres`

## 第五步：配置自动评审定时任务

网站搭好后，需要一个定时任务每天检查哪些投稿超过了 7 天修改期，自动触发 LLM 评审。

### Vercel Cron（如果用 Vercel 部署）

在项目根目录创建 `vercel.json`：

```json
{
  "crons": [
    {
      "path": "/api/cron/lock-and-review",
      "schedule": "0 */6 * * *"
    }
  ]
}
```

这会每 6 小时检查一次。记得在 Vercel 环境变量里设置 `CRON_SECRET`。

### 手动触发

你也可以在浏览器中手动触发评审（比如测试时）：

```bash
curl -X POST http://localhost:3000/api/cron/lock-and-review \
  -H "Authorization: Bearer 你的CRON_SECRET"
```

## 常见问题

**Q: 没有任何 API 密钥怎么办？**
A: 网站仍然可以正常运行，只是 LLM 评审功能无法使用。你可以先投稿、浏览，等拿到密钥后再开启评审。

**Q: 评审结果不满意怎么办？**
A: 可以修改 `src/lib/llm-judge.ts` 中的评审 Prompt，调整评审标准和权重。

**Q: 如何修改网站外观？**
A: 所有页面在 `src/app/` 目录下，组件在 `src/components/` 目录下，使用 Tailwind CSS 样式。
